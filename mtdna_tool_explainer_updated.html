Bio Metadata Audit – Brief System Pipeline and Usage Guide

The BioMetadataAudit Tool is a lightweight pipeline designed to help researchers extract metadata such as geographic origin, sample type (ancient/modern), and optional niche labels (e.g., ethnicity, specific location) from GenBank accession numbers. It supports batch input and produces structured Excel summaries.

Key Steps

  1. Input: One or more GenBank accession numbers are submitted (e.g., via UI, CSV, or text).
  2. Metadata Collection: Using fetch_ncbi_metadata, the pipeline retrieves metadata like country, isolate, collection date, and reference title. If available, supplementary material and full-text articles are parsed using DOI, PubMed, or Google Custom Search.
  3. Text Extraction & Preprocessing:
    • All available documents are parsed and cleaned (tables, paragraphs, overlapping sections).
    • Text is merged into two formats: a smaller chunk and a full all_output.
  4. LLM-based Inference (Gemini + RAG):
    • Chunks are embedded with FAISS and stored for reuse.
    • The Gemini model answers specific queries like predicted country, sample type, and any niche label requested by the user.
  5. Result Structuring:
    • Each output includes predicted fields + explanation text (methods used, quotes, sources).
    • Summarized and saved using save_to_excel.

Output Format

The final output is an Excel file with the following fields:

  • Sample ID
  • Predicted Country and Country Explanation
  • Predicted Sample Type and Sample Type Explanation
  • Sources (links to articles)
  • Time Cost

System Highlights

  • RAG + Gemini integration for improved explanation and transparency
  • Excel export for structured research use
  • Optional ethnic/location/language inference using isolate names
  • Quality check (e.g., fallback on short explanations, low token count)
  • Report Button – After results are displayed, users can submit errors or mismatches using the report text box below the output table
